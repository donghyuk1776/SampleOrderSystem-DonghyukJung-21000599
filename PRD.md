# PRD - 반도체 시료 생산 주문 관리 시스템 (S-Semi)

> 출처: `reference.txt` (reference.pdf 4~23p 요약)

## 1. 배경

가상의 반도체 회사 **S-Semi**는 다양한 반도체 시료(Sample)를 생산하여 연구소, 팹리스(Fabless)
업체, 대학 연구실 등에 납품한다. 시료는 주문이 들어오면 웨이퍼 공정 설비를 통해 제작되고,
검수를 거쳐 고객에게 출고된다.

최근 주문량이 급증하면서 엑셀/메모장 기반 관리로는 아래와 같은 문제가 발생했다.

- 주문 처리 여부를 파악하기 어려움
- 공정 완료 시점을 알 수 없음
- 재고가 충분한데도 불필요한 추가 공정이 발생
- 실수가 잦고 재고·공정 현황을 한눈에 파악하기 어려움

이를 해결하기 위해 콘솔 기반의 **"반도체 시료 생산 주문 관리 시스템"**을 개발한다.

## 2. 시스템 개요

- 콘솔(CLI) 기반으로 동작하며, 담당자가 직접 명령을 입력해 시료 등록과 주문 처리를 수행한다.
- 생산라인은 공장에서 시료 하나를 생산하는 설비 흐름이며, 하나의 생산라인은 시료를 하나씩
  생산하고 주문이 들어온 시료에 대해서만 생산한다.

## 3. 도메인 모델

| 엔티티 | 설명 |
|---|---|
| Sample(시료) | 시스템의 가장 기본 단위. 고유 이름/속성을 가지며 등록된 시료만 주문 가능 |
| Order(주문) | 고객의 시료 구매 요청. 상태를 가지고 전이됨 |
| ProductionLine(생산라인) | 시료를 생산하는 설비. 대기열(생산 큐)을 가짐 |
| Queue(생산 큐) | 생산 대기 주문 목록. FIFO로 스케쥴링 |

### Sample 속성

- 시료ID
- 이름
- 평균 생산시간
- 수율(yield) = 정상적인 시료 수 / 총 생산 시료 수 (예: 100개 중 90개 정상 = 0.9)
- 현재 재고 수량

### Order 속성

- 시료ID
- 고객명
- 주문수량
- 상태(status)

## 4. 주문 상태 흐름

| 상태 | 의미 |
|---|---|
| RESERVED | 주문 접수 |
| REJECTED | 주문 거절 (정상 흐름 외, 모니터링에서 제외) |
| PRODUCING | 주문 승인 완료 + 재고 부족으로 생산 중 |
| CONFIRMED | 주문 승인 완료 + 출고 대기 중 |
| RELEASE | 출고 완료 |

**상태 전이:**

```
RESERVED --(승인, 재고충분)--> CONFIRMED --(출고처리)--> RELEASE
RESERVED --(승인, 재고부족)--> PRODUCING --(생산완료)--> CONFIRMED --(출고처리)--> RELEASE
RESERVED --(거절)--> REJECTED
```

## 5. 기능 명세

### 5.1 메인 메뉴

기능별 선택 화면과 전체 시료 요약 정보를 표시한다.

| 메뉴 | 의미 |
|---|---|
| 시료관리 | 새로운 시료 등록, 목록 조회, 이름 검색 |
| 주문(접수/승인/거절) | 고객 주문 접수 및 생산라인 담당자의 승인·거절 처리 |
| 모니터링 | 상태별 주문 수 및 시료별 재고 현황 확인 |
| 출고처리 | CONFIRMED 상태 주문에 대해 출고 실행 |
| 생산라인 | 현재 생산 중인 시료 및 대기 중인 생산 큐 확인 |

### 5.2 시료관리

- **시료등록**: 새 시료 추가 (시료ID, 이름, 평균 생산시간, 수율 입력)
- **시료조회**: 등록된 모든 시료 목록 + 현재 재고 수량 표시
- **시료검색**: 이름 등 속성으로 특정 시료 검색

### 5.3 시료주문

- **시료예약**: 고객이 원하는 시료/수량을 주문 (시료ID, 고객명, 주문수량 입력)
- 예약 시점의 주문 상태는 `RESERVED`

### 5.4 주문승인/거절

- **접수된 주문목록**: `RESERVED` 상태 주문 목록 표시
- **주문승인**: 재고 상황에 따라 자동 분기
  - 재고 충분 → 즉시 `CONFIRMED`로 전환
  - 재고 부족 → 생산라인에 자동 등록, `PRODUCING`으로 전환
- **주문거절**: 즉시 `REJECTED`로 전환

### 5.5 모니터링

- **주문량확인**: 상태별(`RESERVED`/`CONFIRMED`/`PRODUCING`/`RELEASE`) 목록 확인
  (`REJECTED`는 무효 주문이므로 제외)
- **재고량확인**: 시료별 현재 재고 수량 + 주문 대비 상태
  - 여유: 주문 대비 재고 충분
  - 부족: 주문 대비 재고 수량 부족
  - 고갈: 수량 0

### 5.6 생산라인

- 주문량 대비 부족분을 수율/오차를 고려해 생산
  - 실생산량 = `ceil(부족분 / 수율)`
  - 총생산시간 = `평균 생산시간 × 실생산량`
  - 생산 완료 시 주문 상태 `PRODUCING -> CONFIRMED`
- **생산현황표기**: 현재 생산 중인 시료 정보 표기 (수준은 자율 결정)
- **대기주문확인**: 생산 큐의 대기 목록 표시, 스케쥴링 전략은 **FIFO**

### 5.7 출고처리

- 재고가 충분해진 `CONFIRMED` 주문에 대해 출고 실행
- 주문 상태 `RELEASE`로 전환

## 6. 핵심 계산식 요약

- 수율(yield) = 정상 시료 수 / 총 생산 시료 수
- 실생산량 = `ceil(부족분 / 수율)`
- 총생산시간 = 평균 생산시간 × 실생산량
