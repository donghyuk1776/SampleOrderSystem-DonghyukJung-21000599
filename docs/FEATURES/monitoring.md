# Feature: 모니터링 (Monitoring)

> 상위 문서: [CLAUDE.md](../../CLAUDE.md), [PRD.md](../../PRD.md) §5.5

## 개요

주문에 따른 시료 재고 정보를 모니터링하는 기능. 담당자가 현재 시스템의 상태를 한눈에 파악할
수 있도록 구성한다.

## 요구사항

### 1. 주문량확인

- 현재 상태별(`RESERVED` / `CONFIRMED` / `PRODUCING` / `RELEASE`) 주문 목록/수를 확인한다.
- `REJECTED`는 유효한 주문이 아니므로 집계에서 제외한다.

### 2. 재고량확인

- 각 시료별 현재 재고 수량을 확인한다.
- 주문 대비 재고 수량에 따라 상태를 함께 표기한다.

  | 상태 | 조건 |
  |---|---|
  | 여유 | 재고 수량이 관련 주문 대비 충분한 상태 |
  | 부족 | 재고 수량이 관련 주문 대비 부족한 상태 |
  | 고갈 | 재고 수량이 0인 상태 |

  - "관련 주문"의 기준(예: 해당 시료의 미출고 `CONFIRMED` 주문 수량 합)은 구현 시 명시적으로
    정의하고 문서에 반영한다.

## 예외 처리

- 등록된 시료/주문이 없는 경우 빈 목록/0건으로 정상 출력한다 (에러 아님).

## 테스트 관점

- 각 상태별 주문 수 집계가 실제 데이터와 일치하는지 검증
- `REJECTED` 주문이 집계에서 제외되는지 검증
- 재고 상태(여유/부족/고갈) 분류 경계값 검증 (재고 == 필요량, 재고 == 0)
- 데이터가 없을 때 정상적으로 빈 결과를 출력하는지 검증
