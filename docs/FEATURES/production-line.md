# Feature: 생산라인 (Production Line)

> 상위 문서: [CLAUDE.md](../../CLAUDE.md), [PRD.md](../../PRD.md) §2, §5.6

## 개요

생산라인에 대한 정보를 표시하는 기능. 주문량에 대한 부족분을 생산하되, 수율 및 오차를 고려하여
시료를 생산한다.

## 도메인 모델

- **생산라인(ProductionLine)**: 공장에서 시료 하나를 생산하는 설비 흐름. 하나의 생산라인은
  시료를 하나씩 생산하며, 주문이 들어온 시료에 대해서만 생산한다.
- **생산 큐(Queue)**: 생산 대기 중인 작업(주문)의 대기열. 스케쥴링 전략은 **FIFO**.

## 계산식

- **실생산량** = `ceil(부족분 / 수율)`
- **총생산시간** = `평균 생산시간 × 실생산량`

## 요구사항

### 1. 생산 등록 (주문승인/거절에서 트리거)

- [주문승인](./order-approval-rejection.md)에서 재고 부족으로 `PRODUCING` 전환된 주문은
  생산 큐에 등록된다.
- 등록 시 실생산량과 총생산시간을 계산하여 생산 작업 정보로 저장한다.

### 2. 생산현황표기

- 현재 생산 중인 시료에 대한 정보를 표기한다.
- 표기할 정보 수준은 자율적으로 결정한다 (예: 주문정보, 현재까지의 생산량 등).
- 생산이 완료되면 해당 주문 상태를 `PRODUCING -> CONFIRMED`로 전환하고, 재고를 다음과 같이
  갱신한다: **재고 += 실생산량 − 주문수량**.
  - [주문승인](./order-approval-rejection.md)에서 재고가 충분한 경우 승인 즉시 주문수량만큼
    재고를 차감하는 것과 동일하게, 재고 부족으로 생산을 거친 주문도 `CONFIRMED`로 확정되는
    시점(=이 주문에 재고가 실제로 소진되는 시점)에 주문수량만큼 재고에서 빠져야 한다.
  - 실생산량은 수율 손실을 감안해 부족분보다 크게(`ceil(부족분/수율)`) 생산되므로,
    `실생산량 − 주문수량`은 0 이상이며 이 잉여분이 다른 주문에 쓸 수 있는 재고로 남는다.

### 3. 대기주문확인

- 생산 큐에서 대기 중인 작업 목록을 출력한다.
- 스케쥴링 전략은 **FIFO**: 먼저 등록된 생산 작업이 먼저 처리된다.

## 상태 전이

```
PRODUCING --(생산완료)--> CONFIRMED
```

## 예외 처리

- 수율이 0인 시료는 실생산량 계산 시 0으로 나누기가 발생하므로, [시료관리](./sample-management.md)
  단계에서 사전에 방지되어야 한다.
- 생산 큐가 비어있을 때 대기주문확인 조회 시 빈 목록으로 정상 출력한다.

## 테스트 관점

- 실생산량 계산식(`ceil(부족분 / 수율)`) 검증 (나누어 떨어지지 않는 경우 올림 처리 확인)
- 총생산시간 계산식 검증
- 생산 큐가 FIFO 순서로 처리되는지 검증
- 생산 완료 시 주문 상태 전환(`PRODUCING -> CONFIRMED`) 및 재고 반영 검증
