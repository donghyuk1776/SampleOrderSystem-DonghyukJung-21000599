# Feature: 시료 주문 (Sample Order)

> 상위 문서: [CLAUDE.md](../../CLAUDE.md), [PRD.md](../../PRD.md) §4, §5.3

## 개요

고객이 시료를 요청하면 주문 담당자가 주문을 생성한다. 본 기능은 주문 접수(예약) 단계를
담당하며, 이후 승인/거절은 [order-approval-rejection.md](./order-approval-rejection.md)에서
처리한다.

## 도메인 모델

| 속성 | 타입 | 설명 |
|---|---|---|
| 주문ID | string/int | 주문 고유 식별자 (시스템에서 자동 생성 권장) |
| 시료ID | string/int | 주문 대상 시료 (등록된 시료여야 함) |
| 고객명 | string | 주문 고객 이름 |
| 주문수량 | int | 요청 수량 (1 이상) |
| 상태 | enum | RESERVED / REJECTED / PRODUCING / CONFIRMED / RELEASE |

## 요구사항

### 시료예약

1. 고객이 원하는 시료와 수량을 입력하여 주문을 생성한다.
2. 입력값: 시료ID, 고객명, 주문수량
3. 시료ID는 등록된 시료여야 하며, 존재하지 않는 시료ID로 주문 시 에러 처리한다.
4. 주문수량은 1 이상의 정수여야 한다.
5. 주문 생성 시점의 주문 상태는 `RESERVED`로 설정한다.
6. 생성된 주문은 이후 [주문승인/거절](./order-approval-rejection.md) 흐름을 통해 처리된다.

## 상태 전이 (본 기능이 생성하는 시작 상태)

```
(신규 주문 생성) --> RESERVED
```

## 예외 처리

- 존재하지 않는 시료ID → 주문 생성 거부 및 에러 메시지
- 주문수량이 0 이하이거나 정수가 아닌 값 → 주문 생성 거부 및 에러 메시지
- 고객명 미입력 → 주문 생성 거부 및 에러 메시지

## 테스트 관점

- 정상 입력 시 `RESERVED` 상태의 주문이 생성되는지 검증
- 존재하지 않는 시료ID로 주문 시 거부되는지 검증
- 주문수량이 잘못된 경우 거부되는지 검증
- 동일 시료에 대해 복수 주문이 각각 독립적으로 생성/관리되는지 검증
