# Feature: 주문 승인/거절 (Order Approval/Rejection)

> 상위 문서: [CLAUDE.md](../../CLAUDE.md), [PRD.md](../../PRD.md) §4, §5.4

## 개요

접수된 주문(`RESERVED`) 목록을 확인하고, 생산라인 담당자가 특정 주문에 대해 승인 또는 거절을
처리한다.

## 요구사항

### 1. 접수된 주문목록 조회

- `RESERVED` 상태의 주문 목록을 표시한다.
- 승인/거절 시 입력받는 주문ID는 앞자리 문자 `O`(오)와 숫자 `0`(영)을 헷갈려 입력하는 실수를
  보정한다 (예: `0003` 입력 시 `O003`으로 처리). 그 외 대소문자는 정리하되 형태 자체가 다른
  값은 보정하지 않는다.

### 2. 주문승인

- 접수된 특정 주문(`RESERVED`)에 대해 승인을 처리한다.
- 승인 시 대상 시료의 현재 재고 수량과 주문수량을 비교하여 자동으로 분기한다.

  | 조건 | 처리 |
  |---|---|
  | 재고가 주문수량 이상 (충분) | 재고를 주문수량만큼 차감하고, 주문 상태를 즉시 `CONFIRMED`로 전환 |
  | 재고가 주문수량 미만 (부족) | 부족분에 대해 [생산라인](./production-line.md)에 자동 등록(생산 큐에 추가), 주문 상태를 `PRODUCING`으로 전환 |

- 이미 `RESERVED`가 아닌 주문(예: 이미 승인/거절된 주문)에 대한 재승인 시도는 거부한다.

### 3. 주문거절

- 접수된 특정 주문(`RESERVED`)에 대해 거절을 처리한다.
- 거절 시 즉시 `REJECTED` 상태로 전환한다.
- `REJECTED` 상태는 정상 흐름 외의 상태이며 [모니터링](./monitoring.md)에서 제외된다.

## 상태 전이

```
RESERVED --(승인, 재고충분)--> CONFIRMED
RESERVED --(승인, 재고부족)--> PRODUCING  (생산라인 큐에 등록)
RESERVED --(거절)--> REJECTED
```

## 예외 처리

- 존재하지 않는 주문ID로 승인/거절 시도 → 에러 메시지
- `RESERVED` 상태가 아닌 주문에 대한 승인/거절 시도 → 거부 및 에러 메시지

## 테스트 관점

- 재고 충분 시 승인 → `CONFIRMED` 전환 및 재고 차감 검증
- 재고 부족 시 승인 → `PRODUCING` 전환 및 생산 큐 등록 검증
- 거절 시 → `REJECTED` 전환 검증
- 이미 처리된 주문에 대한 재처리 시도가 거부되는지 검증
- 경계값(재고 == 주문수량) 케이스 검증
