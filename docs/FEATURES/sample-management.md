# Feature: 시료 관리 (Sample Management)

> 상위 문서: [CLAUDE.md](../../CLAUDE.md), [PRD.md](../../PRD.md) §3, §5.2

## 개요

시료(Sample)는 시스템의 가장 기본이 되는 단위이다. 각 시료는 고유한 이름과 속성을 가지며,
시스템에 등록된 시료만 주문이 가능하다. 본 기능은 시료의 등록, 조회, 검색을 담당한다.

## 도메인 모델

| 속성 | 타입 | 설명 |
|---|---|---|
| 시료ID | string/int | 시료 고유 식별자 |
| 이름 | string | 시료 이름 |
| 평균 생산시간 | number | 시료 1개(또는 배치) 생산에 걸리는 평균 시간 |
| 수율 (yield) | float (0~1) | 정상적인 시료 수 / 총 생산 시료 수. 예: 100개 중 90개 정상 = 0.9 |
| 현재 재고 수량 | int | 즉시 출고 가능한 재고 수량 |

## 요구사항

### 1. 시료등록

- 새로운 시료를 시스템에 추가한다.
- 입력값: 시료ID, 이름, 평균 생산시간, 수율
- 시료ID는 중복될 수 없다 (이미 존재하는 시료ID로 등록 시도 시 에러 처리).
- 수율은 0 초과 1 이하 범위로 검증한다 (0인 경우 실생산량 계산 시 0으로 나누기가 발생하므로
  허용하지 않는다).
- 등록 직후 재고 수량은 0으로 초기화한다.

### 2. 시료조회

- 등록된 모든 시료 목록을 표시한다.
- 각 시료의 현재 재고 수량을 함께 표시한다.

### 3. 시료검색

- 이름 등 속성으로 특정 시료를 검색한다.
- 부분 일치 검색을 지원하는 것을 권장한다 (완전 일치만으로 한정하지 않음).
- 검색 결과가 없을 경우 "검색 결과 없음"을 안내한다.

## 예외 처리

- 중복된 시료ID로 등록 시도 → 등록 거부 및 에러 메시지
- 수율이 0 이하이거나 1을 초과하는 값 입력 → 등록 거부 및 에러 메시지
- 존재하지 않는 이름으로 검색 → 빈 결과 안내

## 테스트 관점

- 정상 등록 시 시료 목록에 반영되는지 검증
- 중복 시료ID 등록 시 거부되는지 검증
- 잘못된 수율 값 입력 시 거부되는지 검증
- 이름 검색 시 부분 일치 결과가 반환되는지 검증
- 재고 수량이 다른 기능(생산완료, 출고처리)에 의해 변경된 후 조회 시 최신값이 반영되는지 검증
